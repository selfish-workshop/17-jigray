# StayKo — PRD

## 1. 서비스 개요

**StayKo**는 한국에 거주하는 외국인 유학생이 장기 임대 숙소(원룸, 쉐어하우스, 오피스텔)를 쉽게 찾고 계약할 수 있도록 돕는 중개 플랫폼 데모다. 호스트(집주인)는 매물을 등록하고 예약을 관리하며, 운영진은 호스트에게 수수료를 정산한다. 단순 매물 리스팅이 아니라, **외국인이 겪는 임대 장벽(언어, 보증금, 계약 절차)을 낮추는 UX 실험**이 목적이다.

## 2. 폴더 구조

이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다.

- `planning/` — 기획 (무엇을 만들 것인가)
- `design/` — 디자인 (어떻게 보여줄 것인가)
- `cx/` — CX (누가 어떻게 사용할 것인가)
- `marketing/` — 마케팅 (어떻게 알릴 것인가)
- `strategy/` — 전략 (어떻게 지속 가능하게 만들 것인가)

각 영역의 상세는 해당 폴더에서 작성된다.

## 3. 기획 (Planning)

게스트(외국인 유학생)는 대학교·예산·입주 기간을 기준으로 매물을 검색하고, 온라인으로 예약 신청·계약까지 진행한다. 호스트(집주인)는 매물을 등록하고 예약 승인/거절 후 정산을 받는다. 운영진은 수수료를 계산하고 호스트에게 정산을 실행한다. MVP 범위는 **게스트 매물 검색 → 예약 신청 → 호스트 승인 → 운영진 정산**까지다.

상세는 `planning/` 폴더 참조.

## 4. 디자인 (Design)

깔끔하고 신뢰감 있는 톤. 게스트에게는 Airbnb 스타일의 카드형 매물 리스트를, 호스트에게는 예약 관리 대시보드를, 운영진에게는 정산 관리 테이블을 보여준다. 다국어 전환이 자연스럽고, 외국인이 직관적으로 이해할 수 있는 아이콘·레이블을 사용한다.

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)

타겟 페르소나는 "한국어를 잘 모르는 외국인 유학생"과 "외국인 세입자를 받고 싶지만 소통이 부담스러운 집주인" 두 축이다. 사용자 여정의 핵심은 **매물 발견 → 쉬운 예약 → 안심 계약 → 입주**의 흐름이다.

상세는 `cx/` 폴더 참조. [작성 예정]

## 6. 마케팅 (Marketing)

첫 사용자는 서울·수도권 대학교 외국인 유학생 커뮤니티(Facebook 그룹, 대학 국제처)를 타겟으로 확보한다. "No Korean? No Problem."이라는 핵심 카피로 언어 장벽 해소를 어필한다.

상세는 `marketing/` 폴더 참조. [작성 예정]

## 7. 전략 (Strategy)

데모 단계에서는 수익화보다 유학생·호스트 피드백 확보가 목적이다. 장기적으로는 중개 수수료(계약 성사 시 호스트 측 수수료) 또는 프리미엄 리스팅 광고를 고려한다. 보험·이사·생활서비스 연동까지 확장 가능성을 남겨둔다.

상세는 `strategy/` 폴더 참조. [작성 예정]

| 팀 | 담당 영역 | 작업 폴더 |

| PM (본인) | 전체 기획·의사결정 | 루트 |
